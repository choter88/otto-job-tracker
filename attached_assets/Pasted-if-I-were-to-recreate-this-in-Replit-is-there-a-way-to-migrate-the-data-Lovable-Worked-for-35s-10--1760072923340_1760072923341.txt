CREATE OR REPLACE FUNCTION public.archive_completed_job()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path TO 'public'
AS $function$
BEGIN
  -- Debug: Log the archiving attempt
  RAISE NOTICE 'DEBUG: Archiving job %, status: %, destination: %, OLD status: %',

    INSERT INTO public.archived_jobs (
      id, patient_first_initial, patient_last_name, phone, job_type, 
      status, order_destination, office_id, created_at, updated_at, 
      created_by, assigned_to, status_changed_at, notes,
      is_redo_job, original_job_id, order_id,
      archived_at, archived_by, archive_reason
    )
    VALUES (
      NEW.id, NEW.patient_first_initial, NEW.patient_last_name, NEW.phone, 
      NEW.job_type, NEW.status, NEW.order_destination, NEW.office_id, 
      NEW.created_at, NEW.updated_at, NEW.created_by, NEW.assigned_to, 
      NEW.status_changed_at, NEW.notes,
      NEW.is_redo_job, NEW.original_job_id, NEW.order_id,
      now(), auth.uid(), 'Job completed/cancelled'
    );
    
    RAISE NOTICE 'DEBUG: Archived job %, deleting from jobs table', NEW.id;
    
    -- Delete from main jobs table
    DELETE FROM public.jobs WHERE id = NEW.id;
    
    RAISE NOTICE 'DEBUG: Successfully archived and deleted job %', NEW.id;
    
    -- Return NULL to prevent the update since we deleted the row
    RETURN NULL;
  END IF;
  
  RETURN NEW;
END;
$function$;

4.4.2 Overdue Detection

useAlerts Hook (src/hooks/useAlerts.ts)

Algorithm:

Fetch all active jobs (not completed/cancelled)
For each job:
Calculate daysInStatus = days since status_changed_at
Find matching notification rules for job's current status
If daysInStatus >= rule.max_days: job is overdue
Severity Calculation:
critical: >7 days overdue
high: 3-7 days overdue
medium: 1-3 days overdue
low: Approaching deadline (within 2 days of max)

Edge Function: check-overdue-jobs

Scheduled Execution: Runs periodically (e.g., daily cron)

Process:

Fetch all enabled notification rules
For each rule:
Calculate cutoff date (now - max_days)
Query jobs with status = rule.status AND created_at < cutoff
For each overdue job:
Check if notification already created today
Create in-app notification with severity
If SMS enabled on rule + job has phone:
Check sms_opt_ins for patient consent
Send SMS via send-sms function
Return summary (notifications created, SMS sent)

Call supabase.from('jobs').insert({...})
Database:
Trigger set_order_id(): Generates unique order_id
Trigger log_initial_job_status(): Creates status history record
RLS policy checks user's office_id
Row inserted
Realtime:
Supabase broadcasts INSERT event
Dashboard subscription receives payload
New job added to jobs state
Optional SMS:
If smsStatusTriggers[job_created] enabled:
Check sms_opt_ins for patient phone
Call send-sms edge function
Log attempt in sms_logs
Response:
Toast: "Job created successfully"
Dialog closes, table updates
9.2 Completing a Job

User Actions:

In job table, click status dropdown
Select "Completed"
System Flow:

Client (Dashboard.updateJobStatus):
Optimistic Update: Remove job from jobs array
Call supabase.from('jobs').update({ status: 'completed', status_changed_at: NOW() })
Database:
Trigger trg_10_log_job_status_change: Logs status change
Trigger trg_90_archive_completed_job:
Inserts row into archived_jobs (preserves order_id)
Deletes row from jobs
Returns NULL to prevent UPDATE completion
Realtime:
DELETE event broadcast for jobs table
INSERT event broadcast for archived_jobs table
Dashboard removes job from active list
PastJobsTable adds job to archive list
Optional SMS:
If smsStatusTriggers[completed] enabled:
Send completion SMS to patient
Response:
Toast: "Job status updated successfully"
Job no longer visible in "All Jobs"
Job appears in "Past Jobs"
10. Edge Functions

10.1 send-sms

Path: supabase/functions/send-sms/index.ts

Purpose: Send SMS via Twilio with opt-in validation

Request:


{
  "jobId": "uuid",
  "phone": "5551234567",
  "message": "Your {job_type} is ready",
  "templateVariables": { "job_type": "contacts" }
}
Logic:

Validate required fields
Format phone to E.164
Replace template variables
Check sms_opt_ins for consent
Rate limit check (1 sec minimum)
Call Twilio API
Log to sms_logs
Response:


{
  "success": true,
  "messageSid": "SM...",
  "status": "queued",
  "phone": "+15551234567"
}
Error Handling:

400: Missing fields, no opt-in, Twilio error
429: Rate limit exceeded
500: Unexpected error
10.2 check-overdue-jobs

Path: supabase/functions/check-overdue-jobs/index.ts

Purpose: Scheduled task to detect overdue jobs and send alerts

Trigger: Cron job (e.g., daily at 9 AM)

Logic:

Fetch all enabled notification_rules
For each rule:
Calculate cutoff date (now - max_days)
Query jobs with status = rule.status AND created_at < cutoff
For each overdue job:
Check if already notified today
Create notifications record
If SMS enabled:
Verify opt-in
Send SMS with overdue template
Return summary: { notifications_created: 15, sms_sent: 8 }
Response:


{
  "success": true,
  "notifications_created": 15,
  "sms_sent": 8,
  "message": "Processed 5 rules, created 15 notifications, sent 8 SMS"
}

14. Glossary

Job: An order for glasses/contacts tracked through the system
Office: An optometry practice/clinic
Order Destination: The lab/vendor fulfilling the order
Archival: Moving completed/cancelled jobs to historical storage
Redo Job: Creating a new job based on a previous order
Overdue: Job exceeding max days in current status
RLS: Row-Level Security (Postgres feature)
Edge Function: Serverless backend function (Deno)
TOTP: Time-based One-Time Password (2FA method)
E.164: International phone number format (+1XXXXXXXXXX)
