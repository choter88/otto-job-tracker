Otto Tracker - Product Requirements Document (PRD)
1. Executive Summary

Otto Tracker is a comprehensive job/order management system designed specifically for optometry practices. It enables optical practices to track glasses orders (contacts, prescriptions, sunglasses, etc.) through their complete lifecycle—from order creation to completion. The system features customizable workflows, real-time notifications, SMS communication capabilities, role-based access control, analytics, and a super admin portal for platform-level oversight.

2. System Architecture

2.1 Technology Stack

Frontend: React 18.3.1 with TypeScript
Routing: React Router DOM v6.30.1
Styling: Tailwind CSS with custom animations
UI Components: Radix UI primitives (@radix-ui/react-*)

Forms: React Hook Form v7.61.1 with Zod validation
Date Handling: date-fns v3.6.0
Backend: Supabase (PostgreSQL + Edge Functions)
SMS Provider: Twilio

4.4 Overdue Jobs & Alerts

4.4.1 Notification Rules

NotificationRulesDialog (src/components/NotificationRulesDialog.tsx)

Rule Configuration:

Status: Which job status to monitor
Max Days: Days allowed in that status before overdue
Enabled: Rule active/inactive toggle
SMS Enabled: Send SMS to patient when overdue
SMS Template: Custom message template with variables
Notify Roles: Array of roles to notify (owner, manager, staff)
Notify Users: Array of specific user IDs to notify
Example Rule:


{
  "status": "ordered",
  "max_days": 7,
  "enabled": true,
  "sms_enabled": true,
  "sms_template": "Reminder: Your {job_type} has been {status} for {days} days. Contact us at {office_phone}",
  "notify_roles": ["owner", "manager"]
}
4.4.2 Overdue Detection

useAlerts Hook (src/hooks/useAlerts.ts)

Algorithm:

Fetch all active jobs (not completed/cancelled)
For each job:
Calculate daysInStatus = days since status_changed_at
Find matching notification rules for job's current status
If daysInStatus >= rule.max_days: job is overdue
Severity Calculation:
critical: >7 days overdue
high: 3-7 days overdue
medium: 1-3 days overdue
low: Approaching deadline (within 2 days of max)
OverdueJobsTable (src/components/OverdueJobsTable.tsx)

Displays overdue jobs sorted by severity
Shows days overdue, priority badge, contact info
Inline status change dropdown
"Add Note" action
4.4.3 Automated Overdue Checks

Edge Function: check-overdue-jobs

Scheduled Execution: Runs periodically (e.g., daily cron)

Process:

Fetch all enabled notification rules
For each rule:
Calculate cutoff date (now - max_days)
Query jobs with status = rule.status AND created_at < cutoff
For each overdue job:
Check if notification already created today
Create in-app notification with severity
If SMS enabled on rule + job has phone:
Check sms_opt_ins for patient consent
Send SMS via send-sms function
Return summary (notifications created, SMS sent)

4.5.2 SMS Sending

Edge Function: send-sms (supabase/functions/send-sms/index.ts)

Required Secrets:

TWILIO_ACCOUNT_SID
TWILIO_AUTH_TOKEN
TWILIO_PHONE_NUMBER
Request Body:


{
  "jobId": "uuid",
  "phone": "5551234567",
  "message": "Your {job_type} order is now {status}",
  "templateVariables": {
    "job_type": "contacts",
    "status": "in_progress",
    "office_name": "Smith Eye Care",
    "office_phone": "(555) 123-4567"
  }
}
Process:

Validate phone + message fields
Format phone to E.164 (+1XXXXXXXXXX)
Replace template variables in message
Check Opt-In: Query sms_opt_ins for phone
If not found: Return 400 error (requires opt-in)
Rate Limit: Ensure ≥1 second since last SMS
Send via Twilio API
Log attempt in sms_logs table
SMS Templates (Office Settings):


{
  "ordered": "Your {job_type} order has been placed...",
  "in_progress": "Your {job_type} order is now in progress...",
  "ready_for_pickup": "Great news! Your {job_type} order is ready...",

7. Security Implementation

7.1 Row-Level Security Policies

Pattern 1: Office-Scoped Data


CREATE POLICY "Users can view jobs in their office"
ON jobs FOR SELECT
USING (office_id = get_current_user_office_id());
Pattern 2: Role-Based Access


CREATE POLICY "Owners can update office settings"
ON offices FOR UPDATE
USING (id IN (
  SELECT office_id FROM profiles
  WHERE id = auth.uid() AND role = 'owner'
));
Pattern 3: User-Owned Resources


CREATE POLICY "Users can update their own profile"
ON profiles FOR UPDATE
USING (id = auth.uid());
Pattern 4: Public Insert (e.g., SMS opt-in)


CREATE POLICY "Public can create SMS opt-ins"
ON sms_opt_ins FOR INSERT
WITH CHECK (true);

14. Glossary

Job: An order for glasses/contacts tracked through the system
Office: An optometry practice/clinic
Order Destination: The lab/vendor fulfilling the order
Archival: Moving completed/cancelled jobs to historical storage
Redo Job: Creating a new job based on a previous order
Overdue: Job exceeding max days in current status
RLS: Row-Level Security (Postgres feature)
Edge Function: Serverless backend function (Deno)
TOTP: Time-based One-Time Password (2FA method)
E.164: International phone number format (+1XXXXXXXXXX)
