PRODUCTION DATABASE IMPORT INSTRUCTIONS
========================================

How to Import Supabase Data into Production Database:

SETUP:
1. Click the "Database" icon in your left sidebar (looks like a cylinder)
2. Make sure you're viewing the "Production" database (check dropdown/toggle at top)
3. Click on "SQL runner" or "SQL Playground" tab

IMPORT STEPS (Run these files IN ORDER):

Step 1: Import Offices
   File: prod_01_offices.sql
   - Open the file in your workspace
   - Copy ALL the contents
   - Paste into Production SQL Playground
   - Click "Run"
   - Should see 4 offices imported

Step 2: Import Users  
   File: prod_02_users.sql
   - Copy ALL the contents
   - Paste into Production SQL Playground
   - Click "Run"
   - Should see 5 users imported
   - Passwords:
     * dr.michelle@hello-optometry.com → hello2020
     * contact@hello-optometry.com → hello2020
     * All others → TempPass123!

Step 3: Import Jobs Part 1
   File: prod_03_jobs_part1.sql
   - Copy ALL the contents
   - Paste into Production SQL Playground
   - Click "Run"
   - Should see 30 jobs imported

Step 4: Import Jobs Part 2
   File: prod_04_jobs_part2.sql
   - Copy ALL the contents
   - Paste into Production SQL Playground
   - Click "Run"
   - Should see 28 more jobs (total 58)

Step 5: Import Archived Jobs
   File: prod_05_archived_jobs.sql
   - Copy ALL the contents
   - Paste into Production SQL Playground
   - Click "Run"
   - Should see 44 archived jobs imported

Step 6: Import Other Data
   File: prod_06_other_data.sql
   - Copy ALL the contents
   - Paste into Production SQL Playground
   - Click "Run"
   - Should see comments and notification rules imported

VERIFY EVERYTHING WORKED:

Run this in SQL Playground:
```sql
SELECT 
  (SELECT COUNT(*) FROM offices) as offices,
  (SELECT COUNT(*) FROM users) as users,
  (SELECT COUNT(*) FROM jobs) as jobs,
  (SELECT COUNT(*) FROM archived_jobs) as archived_jobs,
  (SELECT COUNT(*) FROM job_comments) as comments,
  (SELECT COUNT(*) FROM notification_rules) as rules;
```

Expected result:
offices: 4
users: 5
jobs: 58
archived_jobs: 44
comments: 12
rules: 8

DONE! You can now log in to your published app with:

Note: If any step fails, you can run it again. The SQL will overwrite duplicate IDs.
